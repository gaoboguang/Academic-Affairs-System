# 桌面版空库初始化说明（2026-04-27）

本说明适用于本地桌面版首次启动。桌面版默认使用用户数据目录里的新 SQLite 主库，不自动拷贝仓库中的 `data/app.db`。

## 初始化顺序

1. 打开系统设置，确认数据保险箱中的用户数据目录、主库路径、备份目录和 SQLite 完整性。
2. 创建一次本地备份，作为空库起点。
3. 在基础数据中维护学年学期、年级班级、学科、课程类型和常用字典。
4. 导入学生模板 `students_import_template.xlsx`。
5. 导入教师模板 `teachers_import_template.xlsx`，并维护任教关系。
6. 创建考试、配置科目，再导入成绩模板 `exam_scores_import_template.xlsx`。
7. 导入考勤模板 `attendance_import_template.xlsx`。
8. 导入行为模板 `behavior_import_template.xlsx`。
9. 进入分析中心的班主任驾驶舱，检查风险分层和本周行动清单。
10. 在输出中心生成班主任周报或学生跟进包。

## 模板说明

- 考勤状态固定为：正常、迟到、早退、病假、事假、旷课、其他。
- 考勤按同一学生、日期、范围、节次覆盖更新。
- 行为类型固定为：表扬、违纪、谈话、心理关注、安全事件、奖惩、其他。
- 行为记录默认追加；同一文件内同学生、日期、类型、标题重复项会作为提醒并跳过。

## 备份恢复

- 导入真实数据、批量修正或恢复前，先在系统设置创建备份。
- 数据保险箱会显示主库路径、备份目录、SQLite 完整性和恢复演练入口。
- 如果导入后发现大范围错误，优先恢复导入前备份；少量错误优先用模板重新导入覆盖或在业务页面修正。
