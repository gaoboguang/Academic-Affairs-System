## 一、先确认项目定位

这不是一个全新的项目，而是在现有系统基础上继续扩展。

现有系统已经具备：
- 院校库 / 专业库 / 录取库
- 普通生推荐
- 艺体生推荐
- 推荐历史
- 推荐报告导出

所以本轮不要重做推荐中心，而要：
1. 先收尾旧推荐中心的尾项
2. 再把推荐中心升级成“高考志愿报名辅助台”

4. 更新 README

## 三、本轮主线目标

在现有升学中心下新增“高考志愿报名”主入口，并完成以下四个正式能力的建模与落地：

1. 生源地选择
2. 省份规则中心
3. 预估分数推荐模式
4. 目标就业方向推荐

注意：这四项不是页面上多几个筛选字段，而是必须正式落库并可被推荐引擎读取。

## 四、本轮新模块设计要求

### 1）生源地选择（必须新增）
在推荐输入中新增并强制要求：
- origin_province
- gaokao_year
- candidate_type
- subject_combination
- target_batch
- score_input_mode

要求：
- 同一所院校在不同生源地下必须允许出现不同招生计划与录取数据
- 批量推荐时每个学生可以有不同生源地
- 未选生源地时，不允许生成推荐结果

### 2）省份规则中心（必须新增）
新增规则表：
- gaokao_province_rule
- gaokao_score_transform_rule
- gaokao_batch_rule
- gaokao_subject_requirement_dict

必须支持三类模式码：
- new_gaokao_3p3
- new_gaokao_3p1p2
- legacy_wenli

禁止把山东、河北、广东等规则硬编码在前端。

### 3）招生计划库（必须新增）
在现有 admission_record 之外新增：
- enrollment_plan_batch
- enrollment_plan_record
- score_rank_segment

要求：
- 先做 Excel 导入与错误报告
- 原始文件继续保存在 data/uploads/
- 结构化数据入 SQLite
- 必须保留导入批次和来源说明

### 4）预估分数推荐模式（必须新增）
新增 score_input_mode：
- actual_rank
- actual_score
- estimated_score
- estimated_score_and_rank
- score_range
- rank_range

推荐逻辑要求：
- 已有正式位次时，直接按位次推荐
- 只有正式分数时，优先转位次后推荐
- 只有预估分数时，估算位次区间，输出乐观 / 中位 / 保守三档结果
- 所有预估模式结果必须明确标注为模拟推荐

### 5）目标就业方向推荐（必须新增）
新增：
- career_direction
- major_career_mapping
- college_career_tag（如需要）

在推荐输入中新增：
- 第一目标就业方向
- 第二目标就业方向
- 是否接受读研路径
- 是否接受考公 / 考编 / 资格证路径
- 偏好权重（兴趣 / 稳定 / 收入 / 城市 / 学校层级）

推荐引擎要求：
- 录取可行性优先
- 就业方向只做增强排序，不可替代录取规则

### 6）志愿表工作台（建议本轮一起做）
支持：
- 从推荐结果加入志愿表
- 手工排序 / 拖拽排序
- 重复检查
- 志愿上限检查
- 选科匹配检查
- Excel 导出
- 打印预览
