- 使用对象：Codex / 后续开发者
- 目标：在现有“院校库 / 专业库 / 录取库 / 推荐中心”基础上，先收尾既有尾项，再升级为面向全国省份差异化规则的“高考志愿报名辅助台”

## 1. 当前项目状态与本轮目标

### 1.1 当前状态判断

当前项目并不是从零开始，而是：

- 本地运行
- 单用户
- 离线优先
- 中文界面
- 已有院校库 / 专业库 / 录取库
- 已有普通生 / 艺体生推荐
- 已有推荐历史与推荐报告导出

说明推荐中心的基础闭环已经具备，本轮不应重做，而应在现有 M5 基础上升级。

### 1.2 本轮新增目标

本轮在上一版“高考志愿报名”规划基础上，再新增并明确以下能力：

1. **考生生源地可选**
   - 推荐和查询必须以“考生生源地（省份）”为核心前提。
   - 同一所院校在山东招生、在河北招生、在广东招生的数据口径和志愿结构可能完全不同，不能共用一套静态规则。

2. **全国省份高考规则配置化**
   - 将各省高考模式（3+3 / 3+1+2 / 旧高考）、总分口径、赋分方式、选考要求、志愿单位模型纳入规则中心。
   - 规则必须按“省份 + 年份”配置，不能写死在前端。

3. **预估分数推荐模式**
   - 在成绩正式公布前，允许用户按预估分数、预估位次或分数区间进行模拟推荐。
   - 若没有官方当年位次表，则以历史一分一段或近年映射估算位次区间，并给出风险等级。

4. **目标就业方向推荐**
   - 推荐不只考虑“能不能上”，还要考虑“将来想做什么”。
   - 支持按目标就业方向对专业与院校进行二次排序和解释。

5. **志愿表工作台**
   - 推荐结果不是终点，要支持保存、排序、比较、导出预填表 / 志愿表。

## 4.2 省份规则中心（本轮必须新增）

### 目标
把“高考模式、赋分、志愿单位、计划口径、录取口径”全部配置化。

### 为什么必须单独做

因为以下几类差异不能再靠一个 `province` 字段硬扛：

1. 省份考试模式不同：3+3 / 3+1+2 / 旧高考
2. 总分口径不同：如上海 660，海南 900，多数省份 750
3. 赋分口径不同：同是等级赋分，各省转换表和细则并不完全相同
4. 志愿单位不同：
   - 专业（类）+ 学校
   - 院校专业组
   - 院校级 / 专业级
5. 批次与志愿数不同
6. 同一院校在不同省份的计划、专业、组代码、录取位次都不同

- `id`
- `province_code`

- `exam_mode`（`new_gaokao_3p3` / `new_gaokao_3p1p2` / `legacy_wenli`）

- `required_subjects_json`
- `first_choice_subjects_json`

- `score_rule_summary`
- `subject_requirement_mode`
- `volunteer_unit_mode`
- `parallel_rule_mode`
- `note`
- `is_active`

#### `gaokao_score_transform_rule`
用于存放赋分转换规则。

建议字段：
- `id`

- `subject_code`
- `score_mode`（`raw` / `grade_assigned`）
- `grade_table_json`
- `formula_json`

#### `gaokao_batch_rule`
用于存放批次与志愿结构。

建议字段：

## 5.3 规则实现注意事项

1. **不能把“3+1+2”看成所有省份完全相同**
   - 共同点：语数外一般按原始分计入总分，首选物理/历史按原始分计入，总分一般 750。
   - 差异点：再选 2 门的等级赋分比例、转换表、批次组织、志愿表结构、组代码体系可能不同。
   - 因此系统只允许“模式复用”，不允许“规则共用”。

2. **不能把“3+3”看成所有省份完全相同**
   - 北京与其他 3+3 省份在等级赋分细则上并不等价；
   - 上海、海南总分口径又与多数省份不同；
   - 所以 `3+3` 只能作为顶层模式码，细节仍按省份年份配置。

3. **必须保留旧高考兼容层**
   - 西藏与新疆现阶段仍需要支持文史 / 理工逻辑；
   - 系统不能假设全国都已经统一为新高考。

## 7. 预估分数推荐模式（本轮必须新增）

### 7.1 目标
在高考正式出分前，也能让用户做“模拟填报 / 方案预演”。

### 7.2 前端新增输入模式

新增 `score_input_mode`：

- `actual_rank`：已有正式位次
- `actual_score`：已有正式分数
- `estimated_score`：只有预估分数
- `estimated_score_and_rank`：预估分数 + 预估位次
- `score_range`：输入预估分数区间
- `rank_range`：输入预估位次区间

### 7.3 表单字段

- 预估分数
- 预估位次（可选）
- 预估分数下限 / 上限（可选）
- 预估位次下限 / 上限（可选）
- 参考考试（可选：如一模 / 二模 / 三模）
- 是否使用历年映射估算位次
- 结果风险偏好（保守 / 平衡 / 激进）

### 7.4 推荐算法要求

#### 情况 A：已有正式位次
- 直接按位次走推荐主算法

#### 情况 B：只有正式分数
- 若有当年本省分数-位次表，则先换算位次再推荐
- 若无，则按近年同省 score-rank 映射估算，并标记低置信度

#### 情况 C：只有预估分数
- 使用近年同省 score-rank 映射，估算位次区间
- 结果按区间而不是单点推荐
- 输出“乐观 / 中位 / 保守”三档结果

#### 情况 D：预估分数 + 预估位次
- 以位次为主，分数为辅助校验
- 若两者差异异常，提示“输入可能不一致”

### 7.5 输出要求

推荐结果必须明确标识：
- 当前是正式推荐还是模拟推荐
- 位次是正式位次还是估算位次
- 结果置信度
- 是否建议等正式出分后二次确认

### 7.6 推荐解释要求

示例说明：
- “本结果基于山东省 2026 模拟分数 568 分估算位次区间生成，不等同正式志愿建议”
- “本结果使用 2023-2025 年历史 score-rank 映射，建议正式出分后重算”

### 8.5 推荐算法要求

推荐顺序必须是：

1. 先看省份规则能否报
2. 再看分数 / 位次是否匹配
3. 再看选科要求是否满足
4. 再看就业方向匹配度
5. 最后综合地区、费用、院校层级偏好排序

也就是说：
- **就业方向只能增强排序，不能越过录取可行性**
- 不允许把“就业方向匹配”当成单独替代录取规则的依据

### 8.6 输出解释要求

结果中新增：
- 与目标就业方向的匹配等级
- 对应的典型岗位
- 是否常要求读研
- 是否常要求资格证
- 是否属于“可转化路径而非直达路径”

## 10. 推荐算法升级（本轮主线）

## 10.1 新输入维度

新增输入维度如下：

- 生源地省份
- 高考年份
- 成绩输入模式
- 正式分数 / 预估分数 / 位次 / 区间
- 选科组合
- 批次
- 目标地区
- 目标院校层级
- 费用容忍度
- 是否接受调剂
- 第一 / 第二目标就业方向
- 风险偏好

## 10.2 推荐流程（新）

### 第一步：规则装载
根据：
- 省份
- 年份
- 批次
- 考生类别

装载：
- 高考模式
- 赋分口径
- 志愿单位模型
- 选科匹配规则
- 冲稳保阈值模板

### 第二步：成绩标准化
把用户输入转换成标准推荐输入：
- `normalized_score`
- `normalized_rank_low`
- `normalized_rank_high`
- `confidence_level`

### 第三步：候选池过滤
过滤条件：
- 生源地省份匹配
- 年份匹配
- 批次匹配
- 考生类别匹配
- 选科要求匹配
- 艺体方向匹配
- 招生计划存在
- 黑名单 / 白名单

### 第四步：录取可行性评分
建议输出：
- `admission_fit_score`
- `risk_level`
- `bucket_type`（冲 / 稳 / 保 / 高风险）

### 第五步：就业方向与偏好排序
在可报池中再做排序：
- `career_match_score`
- `city_preference_score`
- `school_level_score`
- `fee_score`
- `adjustment_score`

### 第六步：生成志愿表候选
结果分为：
- 推荐清单
- 志愿表草稿
- 导出清单
- 风险提示清单

## 10.3 分类建议

在原“冲稳保”上可增加：

- 极冲
- 冲
- 稳
- 保
- 极保

但 UI 默认仍展示：
- 冲
- 稳
- 保

高级模式可展开更细粒度。

## 17. 风险与边界提醒

1. **2026 年各省最终志愿细则会继续更新**
   - 本轮先建立“规则中心 + 年份版本化”能力，不把未来细则写死
   - 后续每年只增量更新规则数据与计划数据

2. **预估分数本质上不稳定**
   - 预估推荐必须显示为“模拟结果”
   - 不允许伪装成正式志愿建议

3. **就业方向不是绝对映射**
   - 专业到职业方向通常是多对多
   - 只能做“强相关 / 可转化”提示，不能夸大确定性

4. **省份规则差异比想象中大**
   - 同一模式码下也会有不同赋分表、不同志愿结构、不同计划口径
   - 必须按省份年份维护
