# 三、本轮新增重点
## 1）招生计划库
在现有 `college / major / admission_record` 基础上新增招生计划模型与导入流程。
最少支持以下字段：
- 年份
- 省份
- 批次
- 科类/模式
- 院校
- 院校代码
- 专业组编号（可空）
- 专业
- 专业代码
- 计划人数
- 选科要求
- 学费
- 学制
- 培养地点
- 学生类别
- 数据来源
- 导入批次

## 2）省份志愿规则配置
必须配置化，不允许写死。
最少支持：
- 省份
- 年份
- 高考模式（3+3 / 3+1+2 / 文理等）
- 批次定义
- 志愿数上限
- 志愿单位类型（院校 / 院校专业组 / 专业）
- 是否平行志愿
- 是否允许服从调剂
- 特殊批规则

## 3）学生志愿工作台
围绕单个学生形成完整工作流：
- 高考成绩与位次
- 选科组合
- 地区 / 院校层级 / 专业偏好
- 约束项（是否排除民办、中外合作、高收费等）
- 职业与就业方向意向
- 候选池筛选
- 冲稳保推荐
- 志愿表编辑
- 风险提示
- 导出预填表

## 4）推荐引擎升级
保留现有普通生 / 艺体生推荐基础，但要新增：
- 招生计划过滤
- 院校专业组 / 选科要求细粒度过滤
- 更清晰的冲稳保阈值版本
- 推荐解释生成
- 数据样本不足提示
- 计划波动提示
- 章程核验风险提示

## 5）志愿表编辑器
最少支持：
- 从候选池加入志愿表
- 拖拽排序
- 院校代码 / 专业代码显示
- 志愿数上限检查
- 重复检查
- 选科要求检查
- 保底不足提醒
- Excel / HTML / 打印预览导出

## 6）录取跟踪与复盘
新增录取结果导入与复盘能力：
- 导入录取结果
- 记录投档状态 / 最终录取去向
- 对比志愿方案与实际结果
- 统计冲稳保命中情况
- 为后续策略优化留样本

# 四、新增“按目标就业方向推荐”能力（本轮新增重点）
这不是一个简单筛选项，必须落到数据、规则、推荐与导出四层。

## 1）就业方向知识库
新增“职业与就业方向”数据域，至少支持：
- 就业方向名称
- 分类
- 说明
- 常见单位行业
- 常见职位类别
- 是否常要求读研
- 是否常要求资格证/执业证
- 是否属于长培养周期路径
- 风险提示
- 数据来源

## 2）专业就业画像
在专业库上增加就业画像能力，至少支持：
- 常见就业行业
- 常见职位类别
- 就业方向摘要
- 升学/考证提示
- 薪酬指数文本（可选）
- 满意度/发展说明（可选）
- 数据年份
- 数据粒度（专业 / 专业类 / 学科门类）
- 数据来源与可靠性等级

## 3）专业-就业方向映射
新增多对多映射，不允许把某个职业方向只绑定一个专业。
至少支持：
- 就业方向
- 专业
- 关联强度（核心相关 / 强相关 / 一般相关 / 可转化）
- 是否更依赖读研
- 是否更依赖资格证
- 推荐说明
- 是否启用

## 4）学生职业意向采集
在学生志愿工作台中新增：
- 首选就业方向
- 次选就业方向
- 可接受替代方向
- 是否优先稳定性 / 薪酬 / 兴趣 / 长远发展
- 是否接受读研路径
- 是否接受考公 / 考编 / 资格证路径
- 目标就业城市（可选）

## 5）推荐引擎排序增强
推荐逻辑必须保持“录取可行性优先”。
先做可报性过滤和冲稳保分层，再用职业方向匹配做二次排序。

至少新增：
- 目标就业方向与专业映射匹配
- 目标行业与专业就业画像匹配
- 目标岗位类别与专业就业画像匹配
- 是否接受读研/考证/长路径

## 6）推荐解释增强
每条推荐结果至少要解释：
- 为什么这个专业与目标就业方向匹配
- 属于核心相关还是可转化路径
- 是否常需要读研/考证
- 常见就业行业/职位类别
- 当前数据粒度与可靠性
- 是否存在样本不足、计划波动、章程待核验等风险

## 7）报告导出增强
推荐报告新增：
- 目标就业方向摘要
- 推荐专业路径
- 替代路径说明
- 培养周期/读研/考证提示
- 风险提示
