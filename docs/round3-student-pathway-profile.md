# 第三轮 D3 学生升学画像与资格材料缺口

- 日期：2026-04-25
- 范围：窗口 D3，学生升学画像、材料缺口和路径评估可读化
- 后端入口：`/api/gaokao/students/{student_id}/pathway-profile`
- 前端入口：学生详情页 -> `升学画像`

## 目标

D3 在 D1/D2 的路径模型、规则字典和三态评估基础上，把学生个人画像接到老师可操作的页面。页面用于维护山东生源地、考生类型、选科、报名状态、身份、路径接受度、体检限制和资格材料准备情况。

本轮仍只做资格初筛、材料缺口和人工复核提醒，不把单招、综评、春考、艺体、体育、提前批或特殊类型解释成录取概率。

## 本轮完成

- 学生详情页新增 `升学画像` 标签。
- 画像表单支持维护：
  - 生源地、考生类型、考试类型、选科组合。
  - 春考专业类别、艺术类别、体育类别。
  - 高考报名、应届、中职、社会人员、同等学力等身份字段。
  - 是否接受专科、民办、中外合作、省外、提前批、定向服务、面试体检政审。
  - 体检限制、材料备注和 D2 规则会读取的资格材料。
- 保存画像后自动调用路径评估预览，展示各路径状态、缺口、下一步动作。
- 后端评估结果的 `missing_materials_json` 增加中文 `material_label`、`gap_type` 和 `next_action`，前端不需要展示内部材料键。

## 页面读法

- `适合关注`：当前画像已满足关键门槛，但仍需正式报名前核对官方计划和高校章程。
- `可能适合`：没有硬性不满足项，但仍存在政策提醒、人工核验或路径本身只能初筛。
- `信息不足`：缺少画像字段或材料，先补齐页面提示的缺口。
- `不建议`：已有硬性门槛不满足，不建议作为当前主路径。
- `需人工复核`：系统无法自动判断，必须核对官方公告、报名时间、资格材料或高校章程。

## 边界

- 当前没有新增数据库表，继续复用 D1 的 `student_pathway_profile` 和 `student_pathway_evaluation`。
- 当前没有把路径评估做成最终规划方案；D4 可在升学方案中心继续承接画像摘要和路径卡片。
- 材料勾选代表“本地已整理 / 老师已确认”，不是系统自动验证真伪。
- 所有特殊路径仍必须人工复核官方公告和院校章程。
