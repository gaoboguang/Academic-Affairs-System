# 第四轮高考数据库补齐使用说明

- 日期：2026-04-26
- 适用入口：侧栏“高考数据” -> “山东覆盖”
- 适用对象：需要判断当前数据库是否足够可用、还差哪些官方数据的老师

## 1. 先看哪里

打开“高考数据”后，建议先看“山东覆盖”页签。

重点看四块：

1. “健康检查摘要”：判断当前主库是否可读、P0 是否有警告。
2. “数据库补齐结果说明”：查看哪些数据已补齐、哪些仍是部分补齐。
3. “2020-2026 年份覆盖矩阵”：按年份看一分一段、省控线、招生计划、政策参考和章程复核。
4. “剩余缺口”：确认还需要人工下载、官方发布或人工复核的内容。

需要留存材料时，点击“打印覆盖报告”，可在浏览器中打印或保存为 PDF。

## 2. 当前已补齐什么

E6 已补齐 2020-2022 山东官方数据：

| 数据 | 当前结论 |
| --- | --- |
| 一分一段 | 2020-2025 已覆盖 |
| 省控线 / 批次线 | 2020-2025 已覆盖 |
| 官方来源追溯 | 已登记来源、导入批次、本地官方文件和 SHA256 |
| P0 缺口 | 从 6 条降为 4 条 |

这些数据可以支持历史分数线、分数换位次和资格线说明，但仍不能代表 2026 已发布。

## 3. 当前还差什么

| 数据 | 页面状态 | 处理方式 |
| --- | --- | --- |
| 2024 招生计划完整性 | 部分补齐 | 需要继续核验完整官方计划或用户提供官方文件 |
| 2023 招生计划完整文件 | 需人工下载 | 未确认到稳定可复用的官方完整附件前，不导入第三方整理表 |
| 政策参考 | 部分补齐 | 继续把山东官方政策登记为本地文件、SHA256 和政策参考记录 |
| 招生章程限制链 | 需人工复核 | 按高校官网或章程逐校逐专业复核，不能批量标记已完成 |
| 特殊类型专门录取结果 | 只能初筛 | 缺专门录取结果时，春考、艺体、体育、单招、综评不能按完整录取把握理解 |
| 2026 普通类正式计划 | 官方未发布 | 等山东省教育招生考试院发布后再导入 |

## 4. 页面状态怎么理解

| 页面文案 | 含义 |
| --- | --- |
| 已补齐 | 已有可追溯结构化记录，可进入当前健康检查口径 |
| 部分补齐 | 有数据，但数量、年份或来源仍不足以说完整 |
| 官方未发布 | 当前年份官方尚未发布，系统不能提前造数据 |
| 需人工下载 | 需要老师或开发窗口拿到官方原始文件后再登记导入 |
| 需人工复核 | 自动数据只能提示风险，最终要人工核对官方公告或高校章程 |
| 只能初筛 | 可以做资格、材料或方向性提示，不能当作录取概率 |

## 5. 不能做什么

- 不能把单招 / 综评计划限额当成普通类夏季高考正式计划。
- 不能把招生计划补充信息当成完整招生计划。
- 不能用第三方整理表替代官方来源和本地文件 SHA256。
- 不能伪造 2026 一分一段、省控线、普通类计划或投档结果。
- 不能把普通类录取结果复制成春考、艺体、体育、单招或综评专门录取结果。
- 不能批量把章程限制链从“待复核”改成“已确认”。

## 6. 建议复核顺序

1. 每次补数据前先备份 `data/app.db`。
2. 补完后运行 `npm run backend:data-health -- --json`。
3. 打开“高考数据” -> “山东覆盖”，查看 P0 缺口数量是否变化。
4. 打印覆盖报告，留存本轮补齐状态。
5. 对仍标记“官方未发布 / 需人工下载 / 需人工复核”的项目，只记录下一步动作，不包装成已完成。
