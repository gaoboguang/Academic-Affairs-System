# 第四轮学生批量操作使用说明

- 日期：2026-04-26
- 适用入口：侧栏“学生中心”
- 适用对象：需要批量停用学生主档、批量调班、查看调班历史的老师

## 1. 批量删除学生怎么用

批量删除采用“停用学生主档”的方式，不会物理删除成绩、成长档案、附件、推荐记录、志愿草稿、升学画像或路径评估。

推荐操作顺序：

1. 打开“学生中心”。
2. 在学生列表左侧勾选要处理的学生。
3. 点击“批量删除学生”。
4. 先查看预检结果，确认哪些学生可以停用、哪些学生被阻断、每名学生有关联数据。
5. 填写删除原因。
6. 按弹窗里的确认文字完成二次确认。
7. 执行后查看成功、失败和阻断明细。

删除后，默认学生列表不再显示停用学生；历史成绩、成长记录、附件和升学相关数据仍保留，用于审计和追溯。

## 2. 批量调班怎么用

批量调班用于把多个学生转入同一个目标班级，并保留调班批次和逐学生历史。

推荐操作顺序：

1. 打开“学生中心”。
2. 勾选要调班的学生。
3. 点击“批量调班”。
4. 选择目标班级、生效日期，填写调班原因和备注。
5. 如果是跨年级调班，勾选跨年级确认。
6. 点击预检，查看可调班、被阻断和需要注意的学生。
7. 按弹窗里的确认文字执行调班。

系统会记录来源年级班级、目标年级班级、生效日期、调班原因、备注、操作时间和逐学生执行结果。

## 3. 去哪里看调班历史

调班完成后，可以在两个地方查看：

- 学生详情页的“学籍历史”：查看该学生每次班级调整记录。
- 学生成长档案：调班会作为“班级调整”系统事件展示。

调班历史不是老师手写成长记录。它来自系统调班记录，用于说明学生班级流转，不会伪装成人工成长档案内容。

## 4. 常见状态说明

| 状态 | 含义 |
| --- | --- |
| 可执行 | 预检通过，可以继续处理 |
| 被阻断 | 该学生或目标班级不满足条件，不能执行 |
| 需确认 | 存在跨年级、无当前班级等情况，需要老师确认后再执行 |
| 执行成功 | 学生当前班级已更新，并写入调班历史 |
| 执行失败 | 该学生未完成处理，需要看失败原因 |

## 5. 使用边界

- 批量删除是停用，不是物理删除。
- 不要用批量删除清理历史数据；历史数据应保留。
- 调班会更新学生当前班级，同时保留历史。
- 跨年级调班必须先确认。
- 如果预检提示目标班级不存在、学生已停用或学生已在目标班级，应先修正基础数据。

## 6. 建议验收方式

完成一次批量操作后，建议检查：

1. 学生列表是否已刷新。
2. 被删除学生是否默认不再显示。
3. 调班学生当前班级是否更新。
4. 学生详情“学籍历史”是否出现调班记录。
5. 成长档案是否出现“班级调整”系统事件。
