# GitHub 项目介绍改版设计

- 日期：2026-06-23
- 仓库：`gaoboguang/Academic-Affairs-System`
- 受众：学校管理者、教师与开发者，信息权重各占一半

## 目标

把 GitHub 仓库的 About 简介和 README 首页从“开发接手说明 / 阶段日志”改成真正的项目介绍。访客应在首屏快速理解项目用途、核心能力、数据边界和运行形态；开发者应能继续快速找到技术栈、启动命令、目录与验证入口。

## 当前问题

- GitHub About 的 `description` 为空，无法从仓库列表或页面侧栏判断项目用途。
- README 开头很快进入开发接手导航和里程碑，产品价值、适用对象与核心能力不够突出。
- README 中的长篇日期进展会持续过时，并与 `memory-bank/`、专项报告重复。
- 本地运行、学校服务器部署、隐私存储和高考推荐边界没有形成简洁的公开说明。

## 已确认方案

采用“产品与技术并重”的公开落地页结构：前半部分服务学校管理者和教师，后半部分服务开发者。README 不再承担阶段流水账职责，详细进度继续由 `memory-bank/` 和专项文档维护。

### GitHub About

使用以下简介：

> 本地优先的高中教务决策台，统一管理学生、教师、成绩、课表、工作量与升学推荐，支持 Excel 导入导出、SQLite 离线存储及学校服务器受控部署。

本次不设置 Homepage URL，不修改仓库可见性，不添加未经维护的徽章或外部服务链接。

### README 信息架构

1. **项目标题与一句话定位**
   - 明确这是面向高中学校的本地优先教务决策台。
   - 说明可在单机运行，也可部署到学校自有服务器。
2. **项目解决的问题**
   - 把散落在 Excel 中的学生、教师、考试、成绩和升学数据统一管理。
   - 提供可追溯的分析、推荐、报表和备份流程。
3. **核心能力**
   - 学生与成长档案。
   - 考试成绩与多视角分析。
   - 教师、任教、课表、工作量和评教。
   - 高考院校、专业、录取数据与志愿推荐。
   - 报表导出、备份恢复、账号与权限。
4. **本地优先与安全边界**
   - SQLite 和附件默认保存在本机或学校自有服务器。
   - 不依赖必须联网才能运行的云服务。
   - 公网访问必须使用 HTTPS 反向代理和受控账号。
   - 仓库不包含真实学生、成绩、账号、附件或运行数据库。
5. **快速启动**
   - 优先给出 `npm run start:local`。
   - 保留开发模式、停止服务、迁移和管理员初始化命令。
6. **技术架构与目录**
   - 简述 Vue 3、FastAPI、SQLite、Electron 和测试体系。
   - 给出核心目录职责，不展开阶段实现流水账。
7. **开发与文档入口**
   - 链接开发规格、服务器部署、Mac 指南、测试说明和交接文档。
8. **使用边界与当前状态**
   - 说明项目不是开放注册的多租户 SaaS。
   - 高考推荐是本地决策辅助，缺少专门录取结果的数据只能用于初筛，正式填报仍需核对官方规则与招生章程。

## 内容处理规则

- 保留真实存在且仍可用的命令、路径和文档链接。
- 删除 README 中冗长的日期进展和逐项开发记录；不删除其在 `memory-bank/`、Git 历史或专项文档中的来源。
- 不写“生产就绪”“保证录取”“完全安全”等无法由当前验证证明的表述。
- 中文为主，技术名词保留英文官方名称。
- 不加入截图、Logo、状态徽章或新网站，避免扩大本次范围。
